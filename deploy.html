Publishing Your Site

Get the McGinley family site live on Cloudflare Pages — for free

What you'll end up with: A permanent, public URL like https://mcginley-family.pages.dev (or a custom domain if you have one). The site is completely free to host and will never expire as long as your Cloudflare account is active.

How it works

Cloudflare Pages is a free static site hosting service. It works by reading files from a GitHub repository and serving them on the internet. You don't need a server, and you don't need to pay anything. The process has three stages:

  1. Put your site files on GitHub — GitHub is a free online file storage system. You'll create a folder (called a "repository") and upload the site files into it.
  2. Connect Cloudflare Pages to that GitHub repository — Cloudflare will automatically read your files from GitHub and publish them.
  3. Visit your live URL — Your site will be at a *.pages.dev address. Every time you update a file on GitHub, Cloudflare automatically updates the live site.

What you need first

  1. A free GitHub account — Sign up at github.com if you don't already have one. Choose any username; it won't appear in your site URL.
  2. A free Cloudflare account — Sign up at cloudflare.com. The Pages feature is entirely free with no credit card needed.
  3. Your site files — The four files in the mcginley-family-site folder: index.html, family-tree.html, ancestors.html, deploy.html, and style.css.

Step 1 — Upload your files to GitHub

  1. Go to github.com and sign in. Click the green New button (or the + icon in the top right, then "New repository").
  2. Name the repository — for example mcginley-family-site. Leave it set to Public. Tick "Add a README file". Click Create repository.
  3. In your new repository, click the Add file button → Upload files.
  4. Drag all five files (index.html, family-tree.html, ancestors.html, deploy.html, style.css) into the upload area.
  5. Scroll down and click Commit changes. Your files are now on GitHub.
Tip: All five files should be in the root of the repository (not inside a subfolder), so that index.html is the first thing Cloudflare sees.

Step 2 — Connect Cloudflare Pages

  1. Go to cloudflare.com and sign in. In the left sidebar, click Workers & Pages.
  2. Click the Create application button, then choose the Pages tab, then click Connect to Git.
  3. Click Connect GitHub. You'll be taken to GitHub to authorise Cloudflare. Accept the permissions and return to Cloudflare.
  4. Find and select your mcginley-family-site repository. Click Begin setup.
  5. On the "Set up builds and deployments" page:
    • Project name: type something like mcginley-family (this becomes part of your URL)
    • Production branch: leave as main
    • Build settings: leave everything blank — this is a plain HTML site, no build step needed
  6. Click Save and Deploy. Cloudflare will take about 30–60 seconds to process and deploy your site.

Step 3 — Visit your live site

Once deployment completes, Cloudflare will show you your live URL. It will look like:

https://mcginley-family.pages.dev

Share this link with family members anywhere in the world. The site will remain live indefinitely as long as your Cloudflare account is active.

Updating the site later

When you want to add new information or correct something:

  1. Edit the relevant HTML file on your computer (or directly in GitHub's online editor).
  2. In GitHub, go to the file you want to change, click the pencil ✏️ icon to edit it, make your changes, then click Commit changes.
  3. Cloudflare automatically detects the change and re-deploys your site within a minute or two. No further action needed.

Optional: Custom domain name

If you'd like a more personal address like mcginleyfamily.com, you can:

Buy via Cloudflare Easiest

In Cloudflare, go to Domain Registration and search for your preferred name. Cloudflare sells domains at cost (around $10–12/year for .com). Once purchased, it connects to your Pages site automatically.

Buy elsewhere (Namecheap, etc.)

You can buy a domain from any registrar and point it to Cloudflare Pages. In your Pages project, go to Custom domains → Set up a custom domain, and follow the DNS instructions provided.

A custom domain is entirely optional. The free .pages.dev address works perfectly and is permanent.

Common issues

  • "Page not found" on the live site: Make sure index.html is in the root of your repository, not inside a subfolder.
  • The family tree doesn't load: The tree uses D3.js loaded from the internet (cdnjs.cloudflare.com). It requires an internet connection to view — which is always the case when visiting a website.
  • Styles look wrong: Check that style.css was uploaded to the same location as the HTML files (not in a subfolder).
  • Cloudflare says "build failed": This is a plain HTML site with no build step. In the Cloudflare deployment settings, make sure the Build command field is completely empty.
